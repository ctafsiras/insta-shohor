Bonus Section

JavaScript is a single threaded programming language, which means it can only handle a single task at the same time. To run JavaSrcipt every browser uses their own engine where chrome and Node.js use V8 engine. Firstly the engine parse the JavaScript code through the parser and create the AST which means Abstract Syntex Tree. And then it converted to machine code. Finally machine execute the code as we write it.

Though JavaScript is a single threaded language, but it is so smart that it create illusion like it multi threaded language using some hidden tricks which is called event loop. Firstly it sends all the event to Callback Queue and then take one by one and sends to Call Stack. This proccess of sending an event from queue to stack is called JavaScript Event Loop.

Basically Local Storage and Session Storage works exactly in a same way except of few things. By the meaning of local, Local Storage keeps all the data saved locally because of that no data loss happened when respective tab or broser is closed. On the other hand session storage keeps the data temporerily and it disappear when the browser or tab is closed.
